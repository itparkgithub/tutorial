Git workflow

Git workflow works by automatically pulling a remote git repo hosted on github or gitlab or anyother services alike. Pulling means to copy all the files which can then be edited using your personal machine. One the code or project has been edited, we can use git to automatically upload all the changes back to the project repo for analysis and merging.

Git branching

Branching is a way to express how we can use git to seperate all the work being done. For example Joe could be working on front-end of a website while Don works on the backend. If these 2 changes are on the same branch there is no way of differniating between the different works being done if on the same branch. Think of it as a tree. All branches connect to 1 tree to make it whole.

Basic git commands

  • Git clone <url>:  Copies the folder hosted in github to your computer
  • Git status:  Checks what files have been edited and not saved
  • Git add <file name>: Saves your work to the branch
  • Git commit:  Packages the branch to be sent to the github folder. Basically zips the project to be sent to the main folder on github
  • Git push origin <branchname>: Sends the package prepared in the GIT COMMIT command to your git repo on github or gitlab

Setup Git

  1. Create github account here: click me
  2. Create a new empty repo 
  3. Download Git bashâ€‹ here: click me
  4. Config GitBash user name using: git config --global user.name "your_username" / note: this is the username used to setup github account /
  5. Config GitBash user email using: git config --global user.email "email" / note: this is the email used to setup your github account /
  6. Create a new ssh key using: ssh-keygen -t ed25519 -C "your_email@example.com"
  7. Confirm the file location to save your new ssh key: Press <Enter>
  8. Create an SSH passphrase to protect your new SSH key: "input any password" <Enter>
  9. Open your SSH key and COPY the output of the following command: ssh-add ~/.ssh/id_ed25519
  10. Login to Github on browser
  11. Go to SETTINGS -> SSH key -> New SSH key
  12. Pase the SSH key copied from step 9 into the key input and give it a title
  13. You can check your configuration using: git config -l

Clone and update your first project

  1. Retrieve the clone link by: Click CLONE button on top right of your repo and copy 1 of 2 links that pop up
  2. Open Git bash 
  3. Go to desired workspace on computer using: cd "directory" / note check what folders are in directory using: dir/
  4. Clone your repo using: git clone <link copied in step 1>
  5. Go into your newly cloned folder using: cd <repo name>
  6. Switch from the main branch to a new branch using: git checkout -b <new branch name>
  7. Edit or add files into the newly cloned folder
  8. Add the new files or updated files to the branch using: git add <file name> /note you can add all by using a dot (.) instead of <file name>
  9. Commit the changes to package the branch for a push to the main branch using: git commit -m "message to read on github"
  10. Upload your code to the git repo on github using: git push origin <current branch name>
  11. Go to Github 
  12. Go into your project
  13. Go to PULL REQUEST
  14. Create a new PULL REQUEST
  15. Merge PULL REQUEST to completely update your project on git
